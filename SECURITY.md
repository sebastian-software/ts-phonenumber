# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in ts-phonenumber, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities
2. Email us at security@sebastian-software.de
3. Include a detailed description of the vulnerability
4. Include steps to reproduce if possible

## Response Timeline

- We will acknowledge receipt within 48 hours
- We will provide an initial assessment within 7 days
- We will work on a fix and coordinate disclosure

## Security Best Practices

When using this library:

- Keep the library updated to the latest version
- Validate and sanitize user input before passing to the library
- Do not expose internal parsing errors to end users in production
